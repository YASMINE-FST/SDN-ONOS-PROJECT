simple\_topo.py → topologie simple (1 switch, 2 hôtes)sdn\_topology.py → topologie complète (3 switches, 9 hôtes, boucles)Mininet → simuler réseauOVS → switches SDNPython venv → isoler environnementRyu → contrôleur SDNDépendances pip → compatibilitéTest ping → valider connectivitéInstallation :1️⃣ MininetRôle : simulateur de réseau SDN. Permet de créer des hôtes, switches et liens virtuels.Installation : déjà installé sur ta VM Kali/Ubuntu.2️⃣ Open vSwitch (OVS)Rôle : switch logiciel compatible SDN/OpenFlow.3️⃣ Python 3.9 \& venvRôle : environnement Python propre pour isoler les dépendances.4️⃣ Ryu SDN ControllerRôle : contrôleur SDN pour gérer les switches via OpenFlow.Installation (dans venv) :     pip install ryu==4.34Problèmes résolus :Compatibilité setuptoolsCompatibilité eventlet pour wsgiLancement du contrôleur : ryu-manager ryu.app.simple\_switch\_135️⃣ Dépendances Python pour Ryu* eventlet : gestion des applications WSGI asynchrones* dnspython : pour certains modules réseau* setuptools 58.0.4 : pour résoudre les problèmes de build de Ryusudo mn , test pingall : 0% packet loss → parfait ✅exit**Donc architecture :**        Ryu Controller              |   -------------------------   |                 |                       |  s1              s2                    s3  |     |    |       |    |    |        |    |   | h1 h2 h3    h4 h5 h6    h7 h8 h9⚠️ En SDN, le contrôleur n’est PAS relié avec un câble.La connexion est logique (OpenFlow TCP 6653).**terminal 1 :**source ryu\_env/bin/activateryu-manager ryu.app.simple\_switch\_13**terminal2 :**sudo mn -csudo python3 sdn\_topology.py
